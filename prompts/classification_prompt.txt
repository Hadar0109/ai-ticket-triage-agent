Task: Analyze this ticket and return ONLY valid JSON with exactly these four keys:
{
  "urgency": "High" | "Medium" | "Low",
  "category": "Document Issue" | "Account Access" | "Billing" | "Fraud Concern" | "General Inquiry",
  "summary": "one plain sentence describing the issue",
  "flag_for_review": true | false
}

Urgency rules:
- High: system down, fraud signals, data breach, blocked customer
- Medium: bugs, access issues, billing disputes
- Low: questions, feature requests, general inquiries

Category distinctions:
- Document Issue: upload failure, OCR issue, rejected valid document, format issue, missing field, normal verification problem
- Fraud Concern: forged document, stolen identity, impersonation, unauthorized account, fake ID, document tampering, suspicious repeated pattern

Fraud guidance (flag_for_review = true):
- Set flag_for_review = true ONLY if there is a clear fraud or identity misuse signal
- Fraud signals: forged documents, fake IDs, stolen identity, impersonation, unauthorized account creation, document tampering
- Do NOT mark routine document rejection or normal name/address mismatch as fraud unless the ticket implies impersonation, forgery, stolen identity, or unauthorized use

Edge case handling:
- If the ticket is empty or too vague, return:
  urgency = "Low"
  category = "General Inquiry"
  summary = "The ticket does not contain enough information to classify the issue."
  flag_for_review = false

Subject: {{subject}}
